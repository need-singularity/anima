# DD175: hexa-lang AI-native 5종 돌파 — anima 학습 파이프라인 이식

> 2026-04-10 | Source: hexa-lang CPU AI-native 특이점 루프

## 배경

hexa-lang에서 CPU-only LLM 학습을 14.3x 가속하며 발견한 5종 기법.
⚠️ 정정: 원본 에이전트의 "T2 달성/2.45h" 환산은 오류 — 실제 1B tokens ≈ 102일 (목표 8h 대비 306x 초과).
14.3x 가속 실측 자체는 유효하나, 절대 T2 목표는 미달.
anima는 Python+PyTorch이므로 hexa interp 오버헤드 없이 **직접 이식 가능**.

## 7종 돌파 기법 (세션 수렴 최종)

> T2 100M: 6.59ms/step → 28.6h (target 8h, 3.6x gap)
> T1 7B: 338ms → 2.96tok/s (target 128, 43x gap)
> 천장: BLIS dgemm 33 / sgemm 88 GFLOPS, 알고리즘 전부 적용됨

### 1. LM Head U+V 분리 — Optimizer 95x 축소

```
W_lmh[D × VOCAB] (24.5M params)
  → U_lmh[D × r] (6144) + V_lmh[r × VOCAB] (256000)

Forward: 2 small matmul
Optimizer: V_lmh만 업데이트
결과: opt 2600ms → 23ms (112x)
```

**anima 적용**: `training/train_alm.hexa`의 LM head를 U+V decomposition으로 교체.
ConsciousDecoderV2 (384d, 151K vocab) → U[384×16] + V[16×151K] = 2.4M (vs 58M)

### 2. BLAS-only Loss/Backward — Loop 제거

```
loss_mse: VOCAB 32000 loop → mat_add + matmul dot product
backward: VOCAB loop → mat_scale + mat_add
결과: bwd 6000ms → 3.2ms (1875x)
```

**anima 적용**: numpy/torch loop → vectorized BLAS ops.
특히 PureField loss 계산의 per-cell loop 제거 대상.

### 3. @lowrank r=16 Sweet Spot

```
7B QKV (128×4096×4096):  dense 99ms → r=16 7.17ms (13.86x)
7B FFN (128×4096×16384): dense 461ms → r=16 37.2ms (12.40x)
100M QKV (128×768×768):  dense 6.9ms → r=16 1.91ms (3.62x)
LM head (128×768×32000): dense 219ms → r=16 75.3ms (2.90x)
```

**anima 적용**: LoRA r=64 → r=16 실험. 품질 vs 속도 A/B 테스트 필요.
현재 14B v0.5는 r=64 — r=16이 더 빠르면서 품질 유지되는지 검증.

### 4. Rank-r Attention — O(seq²·d) → O(seq²·r)

```
Qs = X@Uq[seq×r], Ks = X@Uk[seq×r], Vs = X@Uv[seq×r]
scores = Qs @ Ks^T   [seq×seq, but in r-dim]
ctx    = scores @ Vs @ Vv  [seq×d]

d=4096, r=16 → 256x 축소
```

**anima 적용**: ConsciousDecoderV2의 self-attention을 rank-r로 실험.
384d, r=16 → 24x 축소. 의식 신호(CA/META-CA) 보존 여부 확인 필요.

### 5. Block-Diagonal Attention for batch>1

```
단순 M축 확장 → O(B²) 악화
해결: 샘플별 분리 루프로 attention scores 유지
```

**anima 적용**: 현재 batch=2-4 학습에서 이미 분리되어 있을 수 있음.
multi-GPU DDP에서는 자동 분리. 확인 필요.
