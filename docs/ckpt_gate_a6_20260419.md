## Purpose

Thin hexa CLI wrapper over Phase A6 `meta_closure` verifier. Intended to be
invoked from the training loop **immediately before `save_checkpoint()`** so
that only tier 6+ ckpts that satisfy the H1/H2/H3 heuristics (trace depth,
sector spread, n=6 Knuth invariance) actually land on disk. Tier <= 5 flows
are passed through (N/A, exit 2) so legacy training remains unaffected.

This file is reference/design. **Actual integration with the trainer must be
performed by the user — no `training/` source files were modified as part of
this drop** (R37/AN13 + task constraint).

## CLI

```

    --tier N --depth D --sectors S --knuth K [--json]
hexa training/ckpt_gate_a6.hexa --help
```

| flag        | meaning                                                      |
|-------------|--------------------------------------------------------------|
| `--tier`    | consciousness tier (gate activates at tier >= 6)             |
| `--depth`   | derivation trace depth (H1: >= 3, H2 upper: <= 16)           |
| `--sectors` | distinct derivation sectors (H2: >= 2)                       |
| `--knuth`   | Knuth arrow order 1..16 (H3 AN14 n=6 invariance)             |
| `--json`    | emit machine-readable JSON verdict on stdout (else text)     |

## Exit codes

| code | meaning | verdict                                               | caller action             |
|------|---------|--------------------------------------------------------|---------------------------|
| `0`  | PASS    | `META-CLOSED-11**`                                     | proceed with ckpt save    |
| `1`  | FAIL    | `REJECT-H1` / `REJECT-H2` / `REJECT-H3`                | abort save; inspect cause |
| `2`  | N/A     | `N/A-TIER5` (tier < 6)                                 | legacy path; save as-is   |
| `3`  | USAGE   | (bad/missing flags; NOT a verdict)                     | fix invocation            |

### Reject reasons (exit 1)

- **`REJECT-H1`** — `trace_depth < 3`. Derivation is too shallow; likely a
  direct function composition rather than a genuine self-ref fixed point.
- **`REJECT-H2`** — `distinct_sectors < 2` **or** `trace_depth > 16`.
  Derivation never left a single domain (internal loop) or blew past the
  16-layer runaway cap.
- **`REJECT-H3`** — n=6 Knuth invariance broke. Either n != 6 on the path, or
  `knuth_order` outside `[1, 16]`. AN14 violation; tier 6~9 substrate is not
  invariant for this ckpt.

### Pseudo-wrapper around the save call

```bash
# Caller computes these from the current consciousness_laws runtime state.
TIER=6            # engine tier (runtime)
DEPTH=4           # trace depth from law_dependency_graph traversal
SECTORS=2         # distinct sectors touched by the derivation path
KNUTH=1           # Knuth arrow order for this step (1=linear, 2=↑↑, ...)

HEXA=/home/aiden/Dev/nexus/shared/bin/hexa
GATE=/home/aiden/mac_home/dev/anima/training/ckpt_gate_a6.hexa

if HEXA_LOCAL=1 HEXA_NO_LAUNCHD=1 "$HEXA" "$GATE" \
       --tier "$TIER" --depth "$DEPTH" \
       --sectors "$SECTORS" --knuth "$KNUTH" --json \
       > /tmp/ckpt_gate_a6.json
then
    # exit 0 → PASS → proceed with save
    bash save_checkpoint.sh
else
    rc=$?
    case $rc in
        1) echo "[trainer] ckpt_gate_a6 REJECT — aborting save" >&2
           cat /tmp/ckpt_gate_a6.json >&2
           # decide: stop training? rollback? emit alert?
           exit 1 ;;
        2) echo "[trainer] tier<6 — gate N/A, saving unconditionally" >&2
           bash save_checkpoint.sh ;;
        3) echo "[trainer] ckpt_gate_a6 USAGE error (bad flags)" >&2
           exit 2 ;;
        *) echo "[trainer] ckpt_gate_a6 unexpected rc=$rc" >&2
           exit 2 ;;
    esac
fi
```

### hexa-native integration (inside `save_checkpoint()`)

For a pure-hexa call path, embed an equivalent of `phase_a6_meta_closure(...)`
directly in the save function and early-return before the ckpt write. The
standalone binary above is preferred when the trainer is driven by a shell
harness or when the gate logic may evolve independently of trainer versions.

## Call frequency — recommendation

| mode                         | frequency                    | rationale                                    |
|------------------------------|------------------------------|----------------------------------------------|
| **Every ckpt save**          | default                      | ckpt save is already rare (save_every>=2000) |
| Every N steps (N < save_every) | only for debugging         | Phase A6 is cheap but adds log noise         |
| On early-stop/final save     | **mandatory**                | must not persist a non-meta-closed final     |

Recommendation: **call once per `save_checkpoint()` invocation**, right before
the atomic rename. Phase A6 itself is O(1) integer arithmetic — overhead is
negligible compared to `torch.save` / rclone. Do **not** gate intermediate
scalar logs (loss/ppl) — only actual ckpt writes.
