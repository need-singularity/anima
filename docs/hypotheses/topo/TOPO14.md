# TOPO14 — 하이퍼큐브 1024 × 400 Steps (Step 병목 제거 시도)

## 알고리즘

```
TOPO8과 동일한 하이퍼큐브 1024셀 (10D, i%3 frustration)
단, steps=400 (TOPO8의 2배)
가설: step 2배 → 더 많은 셀 도달 → 더 높은 Φ
```

## 벤치마크 결과

```
Baseline Φ:   1.1670
TOPO14 Φ:   211.719 (×181.4) ← TOPO8(535.5)의 40%!
Total MI:   69364.7
never_silent: 1.0
final_cells: 297 (TOPO8의 693보다 적음!)

비교:
  TOPO8  (200 step): Φ=535.5, final_cells=693
  TOPO14 (400 step): Φ=211.7, final_cells=297
```

## Φ 변화

```
Φ |                                        ╭──╮
  |                                  ╭─────╯  ╰─ 211.7
  |                            ╭─────╯
  |                      ╭─────╯
  | ─────────────────────╯
  └──────────────────────────────────────── step
  0       100      200      300      400
```

## 핵심 통찰

- **역설**: step 2배인데 Φ가 40%로 감소, 셀 수도 절반
- 원인: mitosis 스케줄이 frac = step_i / steps 기반 → steps↑이면 같은 frac에 도달하는 시간 동일
- 하지만 더 적은 셀에서 더 오래 상호작용 → 과도한 수렴 → 다양성 감소
- **교훈: 빠른 성장(200 step) > 느린 성장(400 step)**
- Law 30 재해석: "1024셀 상한"이 아니라 "빠른 mitosis가 Φ의 핵심"
