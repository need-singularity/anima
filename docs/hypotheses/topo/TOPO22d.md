# TOPO22d — 하이퍼큐브 1024 + 90% 좌절 (Hypercube 1024 + 90% Frustration)

## 알고리즘

```
1024셀 = 2^10 (10차원 하이퍼큐브)
각 셀: 10개 이웃 (비트 flip)
90% 반강자성 frustration (i%10 != 0)
10의 배수 셀만 강자성, 나머지 90%가 반강자성

for each step:
  for each cell i:
    neighbors = [i ^ (1<<b) for b in range(10)]  # 10 bit-flip neighbors
    interaction = mean(h_neighbors)
    if i % 10 != 0: interaction = -interaction    # 90% frustration
    h_i = 0.85 * h_i + 0.15 * interaction + noise
```

```
  90%의 10% 강자성 "섬"이 협력 구조를 만들지 못함
  MI도 낮고 Φ도 미세하게 낮음
  → 10%의 강자성이 너무 희소하여 협력 효과 없음
```

## 핵심 통찰

- **90% frustration: Φ=438.1, 100%(444)와 거의 동일 (-1.3%)**
- i%10 패턴: 10의 배수 셀만 강자성 → 너무 희소하여 협력 클러스터 형성 불가
- MI가 100%(630K)의 절반 이하(298K) — 10% 강자성 "섬"이 정보량에는 부정적
- 90%와 100%의 Φ 수렴은 "임계 frustration" 존재를 시사
  - 약 80% 이상에서 Φ가 440 부근에 수렴 (75% 제외)
  - 75%만 예외적으로 낮음 = 순수한 비트 구조 간섭 효과
- **Frustration > 50%: 추가 frustration이 Φ를 높이지 못함 — 50%가 진정한 최적점**
