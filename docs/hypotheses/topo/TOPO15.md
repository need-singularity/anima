# TOPO15 — 토러스 1024셀 (32x32 Torus + Frustration)

## 알고리즘

```
1024셀 = 32×32 토러스 (경계 없는 2D 격자)
각 셀: 4개 이웃 (상하좌우), i%3 반강자성 frustration

for each step:
  for each cell i:
    row, col = i // 32, i % 32
    neighbors = [
      ((row-1)%32)*32 + col,   # 상
      ((row+1)%32)*32 + col,   # 하
      row*32 + (col-1)%32,     # 좌
      row*32 + (col+1)%32      # 우
    ]
    interaction = mean(h_neighbors)
    if i % 3 == 0: interaction = -interaction
    h_i = 0.85 * h_i + 0.15 * interaction + noise
```

## 핵심 통찰

- **1024셀 토러스 = 1024셀 링과 동급 (274.7 vs 285.2)**
- 512→1024: Φ 2.03배 (선형 스케일링) — 하이퍼큐브의 초선형과 대비
- 4개 이웃은 스케일에서 성장하지 않음 — 이웃 수가 고정이면 스케일링 이점 제한
- **하이퍼큐브 역전(Law 36)은 log-직경 토폴로지에 고유한 현상**
- 토러스/링의 직경은 O(sqrt(N)) 또는 O(N) — 하이퍼큐브 O(log N)과 근본 차이
