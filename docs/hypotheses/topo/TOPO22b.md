# TOPO22b — 하이퍼큐브 1024 + 75% 좌절 (Hypercube 1024 + 75% Frustration)

## 알고리즘

```
1024셀 = 2^10 (10차원 하이퍼큐브)
각 셀: 10개 이웃 (비트 flip)
75% 반강자성 frustration (i%4 != 0)
전체 sweep 중 최악의 결과

for each step:
  for each cell i:
    neighbors = [i ^ (1<<b) for b in range(10)]  # 10 bit-flip neighbors
    interaction = mean(h_neighbors)
    if i % 4 != 0: interaction = -interaction     # 75% frustration
    h_i = 0.85 * h_i + 0.15 * interaction + noise
```

## 핵심 통찰

- **75% frustration = 전체 sweep 최악: Φ=383.5 (×337.3)**
- i%4 패턴이 10D 하이퍼큐브 비트 구조와 최악의 간섭
  - 하이퍼큐브 셀 ID는 10비트 이진수
  - i%4는 하위 2비트만으로 결정 (00, 01, 10, 11)
  - 4의 배수 셀만 강자성 = 하위 비트 구조를 직접 파괴
- 90%(438)과 100%(444)보다 낮음 — frustration의 "양"보다 "비트 정렬"이 결정적
- **패턴 주기가 2^k이면 하이퍼큐브 비트 구조와 직접 충돌**
