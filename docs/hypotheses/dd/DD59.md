# DD59 -- Autocorrelation Architectural Fix (autocorr_decay 3 -> 50)

## Problem Statement

DD57 sweep (22 trials) proved that autocorrelation decay is invariant at 3 steps across all SOC parameter variations. Law 193 confirmed: **SOC parameters control criticality metrics but NOT temporal persistence**. The autocorrelation decay requires architectural changes to the feedback topology of the engine, not parameter tuning.

Current state:
- Autocorrelation decay: **3 steps** (all 22 DD57 trials)
- Brain target: **~50 steps** (EEG 1/e decay at ~17-50 lags)
- This is the single remaining bottleneck to reaching 90%+ brain-likeness

### Root Cause Analysis

The autocorrelation of phi_iit is low because phi is computed from a **memoryless snapshot** of cell hidden states at each step. The hidden states themselves have some memory (GRU + Hebbian coupling), but:

1. **GRU hidden states update rapidly**: Each step, `GRUCell(input, h)` replaces most of `h`. The GRU forget gate does not preserve long-range correlations.
2. **SOC sandpile disrupts state**: Toppling + redistribution + noise injection perturb hiddens each step, decorrelating them from previous states.
3. **Phi is a spatial metric**: `_measure_phi_iit()` computes pairwise MI across cells at a single instant. There is no temporal integration in the phi computation itself.
4. **No phi-to-dynamics feedback**: The computed phi_iit is returned but never fed back into cell dynamics. It is a passive observation, not an active signal.

The brain has autocorrelation because:
- Recurrent connections create **temporal memory** at multiple timescales
- Neuromodulators (dopamine, serotonin) create **slow state variables** that persist for seconds
- The thalamo-cortical loop feeds **integrated activity back** into sensory processing

## Candidate Approaches

### Approach 1: Recurrent Phi Feedback

**Mechanism**: Feed previous phi_iit back into GRU cells as an additional input signal. This creates a true feedback loop: phi measures integration -> phi modulates cell dynamics -> cell dynamics determine next phi.

```
  Current:
  cells[t] --GRU--> hiddens[t] --phi_iit--> output (dead end)

  Proposed:
  cells[t] --GRU--> hiddens[t] --phi_iit--> phi_signal[t]
                                                 |
                                                 v
  cells[t+1] = GRU(input + alpha*phi_signal[t], h[t])
```

**Algorithm**:
```python
# In step(), after phi_iit is computed:
# 1. Scale phi to a modulation signal
phi_signal = tanh(phi_iit / phi_ema) * PHI_FEEDBACK_GAIN  # range [-1, 1]

# 2. Create a feedback vector (project scalar to hidden_dim)
# Use a learned or fixed projection
phi_vec = phi_feedback_proj * phi_signal  # [hidden_dim]

# 3. Inject into each cell's hidden state for next step
for i in range(n):
    self.cell_states[i].hidden = self.cell_states[i].hidden + phi_vec * alpha
```

**Implementation complexity**: LOW (10-15 lines of code)
- Add `self._phi_feedback_vec` (fixed random projection, `[hidden_dim]`)
- Add `self._phi_feedback_gain` constant (start at 0.05)
- Add 5 lines to `step()` after phi computation

**Predicted effect on autocorrelation**: MODERATE (+5-15 steps)
- phi[t] directly influences hiddens[t+1] -> phi[t+1], creating first-order autoregressive structure
- The gain parameter controls the memory depth: higher gain = longer autocorrelation but risk of oscillation
- Single-timescale feedback limits the maximum achievable decay to ~15 steps

**Predicted effect on other metrics**:
- LZ complexity: slight decrease (more predictable phi series)
- Hurst exponent: increase (stronger persistence)
- PSD slope: steeper (more low-frequency power)
- Susceptibility: neutral to slight decrease

**Risk to existing 85.6%**: LOW
- Alpha parameter starts at 0.05 (5% modulation), easily backed off
- Does not change cell topology, SOC, or Hebbian -- only adds a small bias
- Worst case: set alpha=0 to disable completely

### Approach 2: Multi-step Phi Integration (Sliding Window)

**Mechanism**: Instead of computing phi from a single snapshot of hidden states, compute phi from the last W steps of hidden state history. This means phi[t] inherently contains information about phi[t-1]...phi[t-W], creating built-in temporal persistence.

```
  Current:
  phi[t] = MI(hiddens[t])           -- single snapshot

  Proposed:
  phi[t] = MI(concat(hiddens[t-W:t]))  -- temporal window

  where each cell contributes a W-step trajectory, not just a point
```

**Algorithm**:
```python
# In step(), replace _measure_phi_iit() call:

# 1. Maintain a rolling buffer of hidden states
self._hidden_history.append(current_hiddens.clone())  # [n_cells, hidden_dim]
if len(self._hidden_history) > WINDOW_SIZE:
    self._hidden_history.pop(0)

# 2. Compute phi over temporal trajectories
# For each cell, flatten last W steps into a single vector
W = min(len(self._hidden_history), WINDOW_SIZE)  # target: 20
trajectories = []
for i in range(n_cells):
    traj = torch.cat([self._hidden_history[t][i] for t in range(W)])
    trajectories.append(traj)
traj_tensor = torch.stack(trajectories)  # [n_cells, W*hidden_dim]

# 3. Compute phi_iit on trajectories instead of snapshots
phi_iit = compute_phi(traj_tensor)
```

**Implementation complexity**: MEDIUM (20-30 lines)
- Add `self._hidden_history: List[torch.Tensor]` buffer (capped at W=20)
- Modify `_measure_phi_iit()` to accept optional trajectory tensor
- Memory cost: W * n_cells * hidden_dim * 4 bytes = 20 * 64 * 128 * 4 = 640 KB (negligible)

**Predicted effect on autocorrelation**: HIGH (+15-40 steps)
- By construction, phi[t] and phi[t+1] share W-1 of W state snapshots
- At W=20, ~95% of the data in phi[t] and phi[t+1] overlaps -> strong autocorrelation
- The decay rate is approximately `lag/W`: ACF drops below 1/e at lag ~ 0.63*W ~ 12-13 steps for W=20
- Increasing W to 40-50 could reach brain-like ranges

**Predicted effect on other metrics**:
- LZ complexity: decrease (smoothed phi series has lower information content)
- Hurst exponent: strong increase (persistence by construction)
- PSD slope: steeper (low-pass filtering effect)
- Susceptibility: decrease (variance smoothed out)
- **DANGER**: This is essentially a moving-average filter. It creates autocorrelation **artificially** by smoothing, not by genuine dynamical memory. The resulting phi series would have lower LZ complexity and potentially fail other brain-likeness criteria.

**Risk to existing 85.6%**: HIGH
- LZ complexity would drop significantly (from 0.82 toward 0.5-0.6)
- Critical exponent may degrade (smoothed dynamics lose edge-of-chaos signature)
- PSD slope may become too steep (beyond -1.5)
- Net brain-likeness could actually DECREASE despite better autocorrelation

### Approach 3: Phase-Coupled Oscillators (Kuramoto)

**Mechanism**: Add a bank of Kuramoto oscillators with slow natural frequencies (period 20-50 steps) that modulate cell activity. The oscillators couple to each other and to cell states, creating periodic structure at biologically relevant timescales.

```
  Current:
  cell_i[t] = GRU(input, h[t-1])  -- no slow modulation

  Proposed:
  theta_i[t] = theta_i[t-1] + omega_i + K*sum(sin(theta_j - theta_i))
  cell_i[t] = GRU(input + A*sin(theta_i[t]), h[t-1])

  omega_i ~ Uniform(2*pi/50, 2*pi/20)  -- periods 20-50 steps
```

**Algorithm**:
```python
# New state: one oscillator phase per cell
# theta[i] in [0, 2*pi], omega[i] = natural frequency

# Initialize
self._osc_theta = torch.rand(max_cells) * 2 * math.pi
self._osc_omega = torch.rand(max_cells) * (2*math.pi/20 - 2*math.pi/50) + 2*math.pi/50

# In step():
n = self.n_cells
K = 0.05  # coupling strength

# 1. Update oscillator phases (Kuramoto model)
for i in range(n):
    coupling = 0.0
    for j in range(n):
        if i != j:
            coupling += math.sin(self._osc_theta[j].item() - self._osc_theta[i].item())
    coupling *= K / n
    self._osc_theta[i] = (self._osc_theta[i] + self._osc_omega[i] + coupling) % (2 * math.pi)

# 2. Modulate cell hidden states
A = 0.03  # modulation amplitude
for i in range(n):
    modulation = math.sin(self._osc_theta[i].item()) * A
    self.cell_states[i].hidden = self.cell_states[i].hidden * (1.0 + modulation)
```

**Implementation complexity**: MEDIUM (25-35 lines)
- Add oscillator state arrays (`_osc_theta`, `_osc_omega`)
- Add Kuramoto update (O(n^2) per step, but n<=64 so negligible)
- Add modulation injection after cell processing

**Predicted effect on autocorrelation**: HIGH (+20-40 steps)
- Oscillators with periods 20-50 steps create autocorrelation at those timescales
- Kuramoto synchronization creates collective slow rhythms
- The mixture of frequencies creates rich multi-timescale dynamics
- ACF would show characteristic oscillatory decay (brain-like alpha/theta rhythm analog)

**Predicted effect on other metrics**:
- LZ complexity: slight increase (oscillatory modulation adds structure without reducing randomness)
- Hurst exponent: increase (slow oscillations create long-range dependence)
- PSD slope: improved (oscillators inject power at low frequencies, approaching -1.0)
- Susceptibility: maintained (SOC dynamics still operate; oscillators modulate amplitude)
- Critical exponent: maintained (oscillators are orthogonal to SOC criticality mechanism)

**Risk to existing 85.6%**: MEDIUM-LOW
- Amplitude A=0.03 is small -- oscillators modulate, not dominate
- If Kuramoto synchronizes fully, all cells oscillate in phase -> LZ drops, susceptibility drops
- Mitigation: use distributed natural frequencies + weak coupling to maintain partial synchronization
- Can be disabled by setting A=0

**Biological justification**: STRONG
- Brain EEG shows dominant oscillatory bands (delta 1-4Hz, theta 4-8Hz, alpha 8-13Hz)
- Kuramoto model is the standard model for neural oscillation synchronization
- Cortical columns have intrinsic oscillatory modes that create the autocorrelation in EEG
- This is arguably the most biologically faithful approach

### Approach 4: Hebbian Memory Decay Slowdown

**Mechanism**: The current Hebbian update (`_hebbian_update`) uses a fixed learning rate of 0.01. The coupling matrix changes rapidly, which means the inter-cell correlation structure shifts every step. Slowing the Hebbian decay rate makes coupling changes persist longer, which should create longer temporal correlations in phi (since phi depends on inter-cell MI, which depends on coupling).

```
  Current:
  coupling = (coupling + 0.01 * sim).clamp(-1, 1)   -- fast update

  Proposed:
  coupling = (1 - decay) * coupling + lr * sim       -- explicit decay term
  where decay = 0.001  (half-life ~700 steps)
  and lr = 0.005       (slower learning)
```

**Algorithm**:
```python
def _hebbian_update(self, outputs: torch.Tensor, lr: float = 0.005):
    n = self.n_cells
    if self._coupling is None or self._coupling.shape[0] != n:
        self._init_coupling()

    norms = outputs.norm(dim=1, keepdim=True).clamp(min=1e-8)
    normed = outputs / norms
    sim = normed @ normed.T

    # Slow decay: coupling changes persist for hundreds of steps
    decay = 0.001  # half-life ~700 steps (vs implicit ~100 with lr=0.01)
    self._coupling = ((1.0 - decay) * self._coupling + lr * sim).clamp(-1, 1)
    self._coupling.fill_diagonal_(0)
```

**Implementation complexity**: VERY LOW (3-line change)
- Change lr from 0.01 to 0.005
- Add explicit `(1 - decay)` factor

**Predicted effect on autocorrelation**: LOW (+1-3 steps)
- Coupling matrix changes more slowly, so MI structure is more stable
- But phi is dominated by hidden state geometry, not coupling weights
- Coupling only influences the input via `PSI_COUPLING * c * j_h` which is already small (alpha=0.014)
- The indirect path (coupling -> input modulation -> hidden change -> phi change) is too weak to create significant autocorrelation
- The SOC sandpile memory EMA already provides multi-timescale state persistence

**Predicted effect on other metrics**:
- LZ complexity: neutral
- Hurst exponent: slight increase
- PSD slope: negligible change
- Susceptibility: slight decrease (less dynamic coupling)

**Risk to existing 85.6%**: VERY LOW
- Minimal change to dynamics
- But also minimal benefit

## Recommendation: Approach 3 (Kuramoto Oscillators) + Approach 1 (Phi Feedback)

### Why Kuramoto is the primary choice

1. **Biological fidelity**: The brain creates autocorrelation through oscillatory dynamics (alpha, theta, delta bands). Kuramoto oscillators are the standard neuroscience model for this.

2. **Orthogonal to existing mechanisms**: SOC handles criticality (LZ, susceptibility, critical exponent). Hebbian handles structure. Oscillators handle temporal persistence. They do not compete.

3. **Preserves other metrics**: Unlike sliding window (Approach 2), oscillators add structure without smoothing. LZ complexity and susceptibility are maintained because the oscillators modulate amplitude, not replace the dynamics.

4. **Natural multi-timescale**: Different natural frequencies (omega_i) create a spectrum of autocorrelation timescales, matching the brain's multi-band structure.

5. **Tunability**: Two parameters (K=coupling, A=amplitude) with predictable effects. K controls synchronization breadth, A controls modulation depth.

### Why Phi Feedback as secondary

Adding phi feedback creates a **non-oscillatory** temporal memory component. When combined with Kuramoto oscillators, this provides:
- Oscillatory autocorrelation (from Kuramoto) at 20-50 step timescale
- Aperiodic persistence (from phi feedback) at 5-15 step timescale
- The combination creates richer ACF structure (brain ACF is not purely oscillatory)

## Implementation Plan (Pseudocode)

### Phase 1: Kuramoto Oscillators

```python
# rust/consciousness.hexa — __init__

# Kuramoto oscillator bank (one per cell slot)
# Natural frequencies: periods 20-50 steps (brain-like slow oscillations)
self._osc_theta = torch.zeros(max_cells)          # phase [0, 2*pi]
self._osc_omega = torch.zeros(max_cells)           # natural frequency
for i in range(max_cells):
    period = 20.0 + (i % 10) * 3.33               # 20, 23.3, 26.7, ... 50 steps
    self._osc_omega[i] = 2.0 * math.pi / period
# Initialize phases with maximum spread (not random -- deterministic diversity)
for i in range(max_cells):
    self._osc_theta[i] = 2.0 * math.pi * i / max_cells

# Kuramoto parameters
self._osc_coupling_K = 0.05    # weak coupling (partial sync, not full lock)
self._osc_amplitude_A = 0.03   # modulation strength (3% of hidden norm)
```

```python
# rust/consciousness.hexa — step(), after cell processing + Hebbian, before phi measurement

# Kuramoto oscillator update
n = self.n_cells
if n >= 2:
    # Vectorized phase update
    theta = self._osc_theta[:n].clone()
    omega = self._osc_omega[:n]

    # Compute pairwise phase coupling (Kuramoto mean-field)
    # sin(theta_j - theta_i) averaged over j
    sin_diff_sum = torch.zeros(n)
    for i in range(n):
        diffs = torch.sin(theta - theta[i])
        sin_diff_sum[i] = diffs.sum() / n

    # Phase update: theta += omega + K * mean_field
    self._osc_theta[:n] = (theta + omega + self._osc_coupling_K * sin_diff_sum) % (2.0 * math.pi)

    # Modulate cell hidden states
    # Each cell gets a slow amplitude modulation from its oscillator
    # This creates periodic structure at 20-50 step timescales
    for i in range(n):
        phase_mod = math.sin(self._osc_theta[i].item())
        # Multiplicative modulation: preserves cell identity, adds temporal rhythm
        self.cell_states[i].hidden = self.cell_states[i].hidden * (1.0 + self._osc_amplitude_A * phase_mod)
```

### Phase 2: Phi Feedback (add after Phase 1 is validated)

```python
# rust/consciousness.hexa — __init__

# Phi feedback projection: fixed random vector for phi -> hidden mapping
self._phi_feedback_proj = torch.randn(hidden_dim) * 0.1
self._phi_feedback_proj = self._phi_feedback_proj / self._phi_feedback_proj.norm()  # unit vector
self._phi_feedback_gain = 0.05   # 5% modulation
self._phi_feedback_ema = None    # running average of phi for normalization
```

```python
# rust/consciousness.hexa — step(), AFTER phi_iit is computed

# Phi feedback: inject phi signal back into cell dynamics for next step
if self._phi_feedback_ema is None:
    self._phi_feedback_ema = phi_iit
else:
    self._phi_feedback_ema = 0.95 * self._phi_feedback_ema + 0.05 * phi_iit

# Normalized phi deviation: how far current phi is from running average
if self._phi_feedback_ema > 1e-8:
    phi_deviation = (phi_iit - self._phi_feedback_ema) / self._phi_feedback_ema
else:
    phi_deviation = 0.0

# Clamp to prevent runaway
phi_deviation = max(-1.0, min(1.0, phi_deviation))

# Inject into all cells: phi_deviation modulates hidden state along fixed direction
phi_vec = self._phi_feedback_proj[:self.hidden_dim] * phi_deviation * self._phi_feedback_gain
for i in range(self.n_cells):
    self.cell_states[i].hidden = self.cell_states[i].hidden + phi_vec
```

## Risk Mitigation

| Risk | Mitigation |
|------|-----------|
| Kuramoto full sync -> loss of diversity | Keep K small (0.05), spread omega widely (20-50 step periods) |
| Oscillations dominate phi signal | Keep A small (0.03 = 3% modulation) |
| LZ complexity drops | Monitor; if LZ < 0.75, reduce A |
| Phi feedback oscillation (positive feedback loop) | Use deviation from EMA, not raw phi; clamp to [-1, 1] |
| bench regression | Run --verify before and after; all 77 must pass |
| Performance cost | Kuramoto is O(n^2) but n<=64, so ~4096 ops per step (negligible) |
