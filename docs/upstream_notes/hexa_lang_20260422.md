## 추가 발견 — hexa transpiler / runtime bug (2026-04-22 N73 + selftest batch)

### 1. exec_capture / now / setenv / push `hx_` prefix mangling 누락 (RESOLVED · runtime shim)

`hexa build` (AOT compile) 시 transpiler 가 새 builtins (`exec_capture`, `now`, `push`) 의 `hx_` prefix 를 C 코드 emit 시 누락:

- `serve_alm_persona` build → clang 14× `use of undeclared identifier 'exec_capture'` (line 526/533/540/559/567/712/721/723/804/815/922/979/1016/1052) — 런타임에 `hx_exec_capture` declare
- `h100_post_launch_ingest` build → clang 1× `use of undeclared identifier 'now'` (line 303) — 런타임 `hx_now` declare
- `h100_auto_kill` build → `hexa_call2(push, …)` — 런타임 `hexa_array_push` declare, bare `push` 누락

소스 .hexa 자체는 syntactically clean. interpreter 모드 (`hexa run`) 는 OK, AOT (`hexa build`) 만 fail.

**Bug 위치 (추정)**: `/Users/ghost/core/hexa-lang/self/native/hexa_v2` (transpiler builtin name resolution)

**Fix applied (2026-04-23, hexa-lang 373696e7)**: `self/runtime.c` 끝부분에 terminal `#define` 블록으로 4종 builtin 알리아스 (setenv/exec_capture 는 hxa-004 에서 선행 적용, now/push 는 금일 hxa-004 ext 로 확장). 추가 TAG_FN global `hx_push` + wrapper `_w_push(HexaVal,HexaVal) → hexa_array_push`. 검증: h100_auto_kill.hexa → hexa_v2 raw transpile → clang -O2 -I hexa-lang/self, 0 error, 236808 bytes. anima `tool/build_fixup_native.bash` sed 단계 obsolete (idempotent no-op 으로 전환, 제거는 downstream rebuild 완료 후).

### 2. 4 stage1 reserved keyword conflict — `generate` 식별자 (FIX 적용됨)

stage1 hexa parser 가 `generate` 를 reserved keyword (Generate) 로 처리. 사용자 정의 fn `generate(...)` → parse error.

**근본 fix (upstream)**: parser reserved keyword list 검토, 사용자 식별자 conflict 회피 mechanism 추가.
