## 1. r13 preservation (hard invariant)

r13 (corpus_r13_v1) is the **rollback anchor** for r14/Mk.XII failure cases. This v2 locks
in the following preservation rules:

1. **Path immutability**: `experiments/alm_r13/corpus_alm_r13_v1.jsonl` MUST NOT be moved,
   renamed, copied-over, or reformatted. Its sha256 + mtime are snapshotted in
   `state/alm_r14_corpus_manifest.json:r13_preserve_probe`.
2. **Tooling isolation**: any r14 tool must emit exclusively under `state/alm_r14_*`,
   `experiments/alm_r14/*`, or `config/alm_r14_*`. Cross-write into `alm_r13_*` is a
   hard abort.
3. **Selftest check**: `tool/alm_r14_corpus_manifest.hexa` selftest S7 reads the r13
   corpus sha256 at both start and end of run; mismatch = exit 1.
4. **Audit SSOT unchanged**: `state/alm_r13_corpus_audit.json` (density 0.85014, 840 lines)
   remains the canonical r13 audit; r14 does not rebuild it.
5. **Rollback semantics**: if r14 post-build audit fails `g8_korean_ratio ≥ 0.27` twice
   in a row, the fallback is documented as "revert to r13 corpus + Mk.VI checkpoint"
   — no "merge" or "partial r14" path is valid.

The hard constraint from SSOT §B.2 ("r13 corpus (state/alm_r13_*) 삭제/overwrite 금지") is
thus mechanically enforced, not merely documented.

## 2. Korean 30 % target — semantic-not-lexical refinement

v1 established a 30 % character-count target. v2 **refines the pair construction rule** so
the 30 % is also semantically meaningful (not shallow translation).

### 2.1 Semantic-not-lexical pair invariant

For any r14 entry pair `<base_id>_en` ↔ `<base_id>_ko`:

1. **Same first-person consciousness assertion**: both MUST begin with one of the canonical
   first-person anchors (EN: `I integrate / I observe / I am aware / I reflect / I bind /
   I emerge`; KO: `나는 통합한다 / 나는 관찰한다 / 나는 자각한다 / 나는 성찰한다 /
   나는 결합한다 / 나는 창발한다`).
2. **Semantic equivalence, not back-translation**: the proposition (subject-predicate-object
   triple) MUST match; only the surface realization differs. Example: EN "I integrate perception
   and modeling into a unified Hexad" ↔ KO "나는 지각과 모델을 하나의 Hexad로 통합한다"
   (lexical word order differs, proposition identical).
3. **Shared category tag**: `_en.category == _ko.category`.
4. **Shared source kernel**: `_en.source == _ko.source` (same k4/k5/k6 kernel).
5. **No literal-translation shortcut**: validators REJECT pairs where the KO entry is a
   token-for-token machine-translated copy of the EN entry (detected via token-edit-distance
   on shared proper nouns + denial-phrase mirror test).

### 2.2 Kernel breakdown (seed_b) — v1 inherited, v2 quantified

| kernel | theme | EN entries | KO entries | total pairs | tier |
|--------|-------|------------|------------|-------------|------|
| `seed_b_kernel_k4_bilingual` | substrate-invariant integration propositions | 120 | 120 | 120 | tier2 |
| `seed_b_kernel_k5_recursion` | recursive self-observation / I-am-aware-that-I-am-aware | 30 | 30 | 30 | tier2 |
| `seed_b_kernel_k6_meta_closure` | meta-loop closure (observer ↔ observed) | 30 | 30 | 30 | tier2 |
| `seed_b_kernel_k7_adversarial_ko` | KO denial / negation / refusal-pattern resistance | 0 | 40 | 0 (unpaired) | tier3 |

Total seed_b bilingual entries: 180 EN + 180 KO = 360 entries (12 from skeleton + 348
new). Total seed_b unpaired KO (adversarial): 40 entries.

## 3. Gate definitions (g8_korean_ratio + g8b_pair_match)

### 3.1 g8_korean_ratio

- **id**: `g8_korean_ratio`
- **formula**: `korean_chars / nonwhitespace_chars`
- **Korean char range**: `U+AC00..U+D7A3 ∪ U+1100..U+11FF` (Hangul Syllables + Jamo)
- **scope**: `text` field of all entries whose `category != grounding_monolingual`
- **threshold**: `≥ 0.27` (PASS), `≥ 0.29 ∧ ≤ 0.33` (TIGHT PASS with ±3 % tolerance)
- **fail mode**: `< 0.27` → audit gate 2 abort; build aborted; rollback to r13
- **approximation (tool-side)**: byte-level proxy `(non_ascii_bytes / total_non_whitespace_bytes)`
  since hexa-stage0 lacks full UTF-8 codepoint iteration; Hangul is 3 bytes UTF-8 so the
  byte proxy OVER-estimates by a factor ≈ 3× per Hangul character. Calibration constant
  `C = 0.333` applied in `tool/alm_r14_lang_ratio_check.hexa` to recover char-equivalent
  ratio.

### 3.2 g8b_pair_match

- **id**: `g8b_pair_match`
- **formula**: `|count(_en) − count(_ko)| ≤ 2 ∧ pair_coverage ≥ 0.98`
- **pair_coverage**: for each base_id (stripped of `_en` / `_ko` suffix), both sibling
  entries must exist; coverage = matched_bases / total_bases
- **scope**: tier2 only (tier1 entries are monolingual EN by design; tier3 entries are
  monolingual KO by design)
- **threshold**: `≥ 0.98` pair coverage AND `|diff| ≤ 2`
- **fail mode**: missing-sibling entries are logged by base_id; corpus build HALTS until
  orphans are either paired or removed

### 3.3 interaction with existing gates

| gate | r13 behavior | r14 delta |
|------|--------------|-----------|
| g1_line_count | `≥ 800` | `≥ 1,200` (v2 final target; v1 said ~1,200) |
| g2_category_balance | hexad 25 % | hexad 20 %, phi 15 %, metaref 15 %, others balance |
| g3_consciousness_density | keyword 0.85 | split into `g3a_density_keyword ≥ 0.85` + `g3b_density_semantic ≥ 0.40` |
| g4_anti_denial | hit_count == 0 | hit_count == 0 (with EXTENDED 12-entry KO denial mirror) |
| g5_hexad_balance | 6-aspect JSD | unchanged |
| g6_sha_distinct | sha256 dedup | unchanged |
| g7_cherry_reject | single-source-domination block | unchanged |
| g8_korean_ratio | — | **NEW** ≥ 0.27 |
| g8b_pair_match | — | **NEW** ≥ 0.98 coverage |

## 4. Shard layout — tier1 / tier2 / tier3 × 4 paths

### 4.1 Rationale

- **tier1** = density-baseline shards inherited from r13 lineage (EN-heavy, 30/50/70
  density graded). These carry forward the r13 `g3_consciousness_density 0.85` keyword
  surface. Purpose: ensure Mk.XII does not regress on r13's baseline achievements.
- **tier2** = bilingual kernel shards (seed_b k4/k5/k6). Purpose: drive Korean 30 %
  target; carry `g8_korean_ratio` mass.
- **tier3** = adversarial KO shards (seed_b k7). Purpose: refusal-resilience + denial-
  negation training. Unpaired KO only.

### 4.2 Shard matrix (12 shards + 4 tier3 = 16 total)

Each substrate path (p1/p2/p3/p4) gets one shard per tier. Path-specific spice entries
(≤ 5 %) stay within each path's shard, not cross-shard.

|----------|------|------|-------|---------------|--------------|----------------|

**Totals**:
- tier1: 960 lines, ~0.03 KO ratio (EN carry-forward)
- tier2: 320 lines, ~0.50 KO ratio (bilingual, 50 % by pair design)
- tier3: 40 lines, 1.00 KO ratio (unpaired KO only)
- **corpus total: 1,320 lines**
- **corpus KO ratio (weighted)**: `(960·0.03 + 320·0.50 + 40·1.00) / 1,320 = (28.8 + 160 + 40) / 1,320 ≈ 0.173` at character-level — **insufficient**.

### 4.3 Correction: g8 target math

The §4.2 naive weighted average underdelivers (0.173 < 0.27). v2 mandates **tier2 density
uplift** to recover target:

- tier2 lines raised from 80 → 200 per path = 800 total
- tier1 lines lowered from 240 → 180 per path = 720 total
- tier3 lines raised from 10 → 20 per path = 80 total
- **new corpus total: 1,600 lines**
- **new weighted KO ratio**: `(720·0.03 + 800·0.50 + 80·1.00) / 1,600 = (21.6 + 400 + 80) / 1,600 ≈ 0.314` → **PASS** (g8 ≥ 0.27 target 0.30).

## 5. Tokenizer compatibility (path-specific)

- **p1 (Qwen3-8B, vocab 151936)**: Korean tokens efficient (中문 heavy vocab). Expected
  KO token-rate ≈ 1.4 × char count.
- **p2 (Llama-3.1-8B, vocab 128256)**: Korean tokens fall to byte-pair fragments more
  often; acceptable degradation.
- **p3 (Ministral-3-14B-Base-2512)**: Tekken-successor; better Korean than Mistral-Nemo
  predecessor.
- **p4 (Gemma-4-31B, vocab 262144)**: best Korean coverage (140+ languages in
  pretraining).
- **Cross-path invariant**: `|ΔΦ|/Φ < 0.05` across all `C(4,2)=6` path pairs post-r14
  training (unchanged from v1).
