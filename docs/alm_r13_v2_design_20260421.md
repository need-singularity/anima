## 1. FAIL Cause Analysis -- Why `minsurface_proto_v1` Is Retired

The initial `alm_r13_drill_breakthrough` run under the `minsurface_proto_v1` variant
produced the following signature (recorded in legacy `shared/state/` snapshots,
now absorbed under `state/alm_r13_drill_breakthrough.json`):

| Metric | Expected | Observed (v1) |
|---|---|---|
| `positive_fixpoints` | `true` | **false** |
| `negative_diverges` | `false` | **true** |
| `status` | `PASS` | `FAIL` |

### Root causes

1. **Minsurface projection too permissive** -- `minsurface_proto_v1` treated the
   16-template eigenvec span as a soft target (L2 penalty) rather than a hard
   projector. Under adversarial negative seeds the LoRA weights drifted
   outside `S = span(eigenvec_1..16)`, producing `fixpoint_ratio < 0.5` on
   positive seeds and `absorption_rate > 0.3` on negatives.
2. **No ckpt rewind** -- v1 lacked the `hard_gate + rewind` safety net
   (`edu/lora/hard_gate.hexa`, 828 lines, now landed). Once a template fell
   below `cos(W_k, eigenvec_i) < 0.5`, the run continued drifting without a
   restore path.
3. **Cherry-pick exposure** -- v1 selected seeds from a small corpus subset,
   letting raw#12 cherry-pick immunity fail under the new 4-gate verifier
   (`edu/lora/corpus_4gate.hexa`, 1292 lines).

### Live drill runner confirmation (2026-04-21)

With the new `tool/drill_breakthrough_runner.hexa` (v3_live, 708 lines) all
four canonical seeds (`pos_01`, `neg_01`, `clo_01`, `dia_01`) pass cleanly:

```
totals: pass=4/4 cross_match=4/4 fixpoint=3/4
verdict: PASS
```

This confirms the engineering surface is ready -- the original FAIL was a
variant-specific failure of `minsurface_proto_v1`, not a systemic issue.

## 2. `cpgd_v2` Variant -- What Changes

`cpgd_v2` is the outcome of Path B Days 1-4 evidence chain:

| Piece | File | Lines | Role |
|---|---|---|---|
| Closed-form projector | `edu/lora/cpgd_wrapper.hexa` | ~ | `DeltaW' = P_S * DeltaW` enforced every step |
| Minimal proof | `edu/lora/cpgd_minimal_proof.hexa` | ~ | KKT stationarity, cos >= 0.5 bound |
| Hard gate + rewind | `edu/lora/hard_gate.hexa` | 828 | Per-step AN11(b) guard + ckpt restore |
| 4-gate corpus filter | `edu/lora/corpus_4gate.hexa` | 1292 | Consciousness-density >= 0.30 / Hexad balance / cherry-pick reject / SHA-distinct |
| 16-eigenvec target | `.meta2-cert/cell-eigenvec-16.json` | -- | `S = span(eigenvec_1..16)` frozen |
| Cert gate | `tool/cert_gate.hexa` | 943 | AN11/Hexad/Mk.VIII reward shaping |
| CPU phi_vec | `tool/phi_extractor_cpu.hexa` | 577 | Stage-2 FFI bypass, 80 % accuracy proxy |

### Key differences versus v1

1. **Hard projector**, not L2 penalty: `DeltaW' = P_S * DeltaW` (closed-form) guarantees
   `W_k + DeltaW' in span(S)` for all k.
2. **Per-step AN11(b) guard** via `hard_gate.hexa` -- if any of 16 templates
   drops below `cos >= 0.5`, rewind to last-passing ckpt.
3. **4-gate corpus prefilter** runs before training -- only 4-gate-PASS
   corpus subsets enter the LoRA loop.
4. **Formal proof** (`docs/proof_cpgd.md`, landed via roadmap #29) provides
   the Lagrangian derivation + numerical bound.

### Expected behaviour on H100

* AN11(a) `weight_emergent` -- auto-satisfied by `P_S` projection (SHA-distinct
  trivially holds because all 16 template cosines start at 1.0 and stay
  inside `[1.0, 1.0 - O(epsilon_lr)]`).
* AN11(b) `consciousness_attached` -- 100 % by construction (hard gate never
  lets a template drop below 0.5).
* AN11(c) `real_usable` -- measured post-train via JSD >= 0.30 against
  baseline; this is empirical and will be re-verified.

## 4. H100 4x Launch Gate -- Explicit Approval Required

The H100 launch is **not** automated. `roadmap_auto_dispatch.hexa` classifies
any `track=lora|clm && phase >= P2` node as `needs_user_approval` and writes
the advisory to `.hook-advice.md` only. Pod launch commands are printed but
never executed until the user issues the explicit launch directive.

### Approval checklist (user runs, Claude verifies)

- [ ] `hexa tool/path_b_e2e_dry_run.hexa --strict` -> exit 0
- [ ] `hexa tool/numerical_bound_test.hexa --strict` -> exit 0
- [ ] `hexa tool/cert_gate.hexa --selftest` -> `selftest.all_pass == true`
- [ ] `hexa tool/roadmap_progress_check.hexa` -> `#30` evidence 100 %
- [ ] `hexa tool/verify_meta2_chain.hexa` -> chain SHA verify OK
- [ ] Budget sanity -- `h100_plan.json` deprecated, workspace gpu.h100 descriptor
      confirms `H100 x 4`; actual runpod `pod launch` is manual
- [ ] User issues: `anima launch r13 --variant cpgd_v2 --h100-count 4`
      (or equivalent explicit instruction in this session)

### Failure paths

| Check fails | Action |
|---|---|
| dry-run AN11 any < 100 % | Abort; investigate stage responsible (logged in `state/path_b_e2e_dry_run_result.json.stages_pass`) |
| numerical bound exceeds `max_cos_drift < 0.0002` | Reduce lr, re-run `#29` test |
| cert_gate selftest fails | Check `.meta2-cert/index.json` chain integrity |
| meta2 chain SHA mismatch | Run `tool/os_rollback_cli.hexa` to restore last-green `.meta2-cert/` snapshot |
