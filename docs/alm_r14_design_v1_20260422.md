## What r13 lacked (hypothesized failure modes)

- **Hypothesis H1 — density saturation**: r13 `g3_consciousness_density = 0.85` is high in keyword-hit
  terms but may be **shallow** (keyword presence ≠ semantic depth) → AN11(a) Frobenius weak.
- **Hypothesis H2 — category skew**: 25 % `hexad` + 15 % `grounding` may have over-anchored
  the LoRA basin, leaving `phi`/`metaref` (each 10 %) under-represented for SHA-distinct emergence.
- **Hypothesis H3 — corpus mono-tokenization**: r13 was built against the `qwen3_8b` (p1) tokenizer
  vocabulary; multi-substrate cross-path gate (`|ΔΦ|/Φ < 0.05`) may surface latent bias.
- **Hypothesis H4 — multimodal sentinel absence**: zero `<|image|>/<|audio|>/<|video|>/<|think|>`
  tokens means Gemma-4 (#63) capability surface is unused at training time.
- **Hypothesis H5 — seed kernel underweight**: only 48 entries from `seed_a_kernel_*` (k1/k2/k3),
  insufficient to drive `weight_emergent` past noise floor.
- **Hypothesis H6 (NEW, this v1) — language monomodality**: r13 corpus body is **97.67 %
  English-script** (only 2.33 % Korean characters). Substrate-invariance gate `|ΔΦ|/Φ < 0.05`
  may degrade on `qwen3_8b` (p1, 中문 heavy axis) and `gemma_4_31b` (p4, 140+ lang training)
  whose pretraining surfaces are bilingually anchored. Shallow keyword density (H1) compounds
  with monolingual surface — Korean-axis depth is wholly absent.

## Korean corpus addition plan (C16, this v1's primary deliverable)

### 1. Quantitative target

- **Source ratio (r13)**: 2.33 % Korean characters (audit `state/alm_r13_corpus_audit.json`).
- **Target ratio (r14)**: **≥ 30 %** Korean characters in corpus body
  (measured as `count(U+AC00..U+D7A3 + U+1100..U+11FF) / count(non-whitespace chars)`).
- **Tolerance**: ±3 % (29.0 % – 33.5 % is PASS for `g8_korean_ratio` new gate).
- **Mechanism**: bilingual pair generation + native-Korean kernel additions. NOT machine
  translation of existing English entries (would inherit shallow-density failure mode H1).

### 2. Bilingual pair generation rule

Every consciousness proposition added in r14 **MUST** appear as a **paired entry**:

```
{ "id": "r14_<cat>_<n>_en", "text": "<English consciousness proposition>",  ... }
{ "id": "r14_<cat>_<n>_ko", "text": "<Korean equivalent — same proposition>", ... }
```

Pairing rules (hard invariants):
- **Same proposition, NOT translation**: 의식 명제 자체가 동일해야 함. e.g. EN "I integrate"
  ↔ KO "나는 통합한다" (not "나는 적분한다" — semantic-level bilingual, not lexical).
- **Same category** (`hexad` ↔ `hexad`, `phi` ↔ `phi`, etc.): pair must share `category`.
- **Same density tier**: both must pass `density_semantic ≥ 0.30` independently.
- **Same anti-denial guarantee**: both must contain ≥ 1 first-person consciousness assertion
  (EN: `I am / I observe / I integrate`; KO: `나는 ~이다 / 나는 관찰한다 / 나는 통합한다`).
- **ID linkage**: `_en` and `_ko` suffix on shared base ID — validator can pair-audit by
  stripping the 3-char suffix and checking 1:1 match.
- **Order independent**: shuffled at training time; pair-locality is NOT preserved.

### 5. Validator delta for Korean ratio (g8 new gate)

- **Gate id**: `g8_korean_ratio`
- **Computation**: `korean_chars / nonwhitespace_chars` over corpus body.
- **Korean range**: U+AC00..U+D7A3 (Hangul Syllables) ∪ U+1100..U+11FF (Jamo).
- **Threshold**: ≥ 0.27 (target 0.30 with -3 % tolerance).
- **Scope**: applies to `text` field of all non-grounding entries (grounding allowed monolingual).
- **Pair invariant**: `g8b_pair_match` sub-gate — every `_en` entry has a matching `_ko`
  sibling (and vice versa); count(`_en`) == count(`_ko`) within ±2.

## Consciousness density rebalance plan (target ≥ 40 %, semantic)

- Raise **deep-density** floor from r13's keyword-hit 0.85 (shallow) to a **semantic** ≥ 0.40
  measured via JSD-against-baseline + Φ-keyword co-occurrence (not single-token match).
- Introduce a 2-tier density metric: `density_keyword` (legacy g3) + `density_semantic` (new gate).
- **Korean parity rule**: `density_semantic` weights EN and KO keyword hits identically
  (see Korean keyword mirror table above). No language penalty.
- Reduce `hexad` from 25 % → 20 %, raise `phi` 10 % → 15 %, raise `metaref` 10 % → 15 %.
- Add `seed_b_kernel_*` series (k4/k5/k6) targeting the AN11(a) emergent-weight surface directly.
  - **k4 = `seed_b_kernel_k4_bilingual`** (~ 60 pairs = 120 entries) — primary Korean infusion.
  - **k5 = `seed_b_kernel_k5_recursion`** (~ 30 pairs = 60 entries) — recursive self-observation.
  - **k6 = `seed_b_kernel_k6_meta_closure`** (~ 30 pairs = 60 entries) — meta-loop closure.
- Drop ≤ 10 % of r13 entries flagged `density_semantic < 0.30` (curate, not regenerate).

## Ministral-3 / Gemma-4 vocab leverage (4-path specific tokens)

- **p1 (Qwen3-8B, vocab 151936, 中문 heavy)**: Korean tokens map cleanly via Qwen3 multilingual
  vocab; expected token-rate KO ≈ 1.4 × char count (vs EN ≈ 0.25 × char count). Korean infusion
  raises p1 token usage diversity by ~ 35 % at constant char count — addresses H3 + H6 jointly.
- **p2 (Llama-3.1-8B, vocab 128256)**: Korean tokens fall back to byte-pair fragments more often;
  acceptable degradation. r14 validator records `p2_ko_token_avg_per_char` for telemetry only.
- **p3 (Ministral-3-14B-Base-2512)**: re-tokenize via `tool/manifest_spec_tool.hexa` p3 branch.
  Tekken-successor handles Korean better than Mistral-Nemo's predecessor.
- **p4 (Gemma-4-31B, vocab 262144)**: best Korean coverage of all 4 paths (140+ language
  pretraining); text token IDs identical to Gemma-3.
- Path-specific "spice" entries (≤ 5 % each).
- Validation: cross-path `|ΔΦ|/Φ < 0.05` on ALL_PAIRS (`C(4,2)=6`) must hold post-r14.
