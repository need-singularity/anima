## §4. Cycle 4 Prospective Corpus (Agent 3's 8-Task Design)

Pre-registered before any data is observed. The corpus is **eight independent prediction tasks** drawn from canonical CA / sequence / boolean / random-walk literature:

1. **Wolfram rule 30** — chaotic 1-D CA, predict t+1 cell from window-5 history.
2. **Wolfram rule 90** — XOR Sierpinski CA, additive elementary rule.
3. **Wolfram rule 110** — Turing-complete elementary CA (cycle-3 task, retained).
4. **Wolfram rule 184** — traffic-flow CA, conservation-law class.
5. **Conway Game of Life** — 2-D CA, predict center cell from 5×5 neighborhood.
6. **Integer sequences** — OEIS-style next-term prediction (Fibonacci, Catalan, primes mod-k).
7. **Boolean function learning** — random k-CNF target, predict output from input bits.
8. **Random-walk prediction** — biased 1-D walker next-step, signal-to-noise gradient.

Each task: 10k train / 1k held-out test. Per-task metric: top-1 accuracy. Aggregation: **Stouffer's z** combination of per-task z-scores (CA(5) advantage vs Transformer baseline). Pre-registered claim-check threshold: 81% aggregate advantage → corroboration; <50% → falsification; in-between → suspended.

## §5. Real Transformer Baseline

Cycle-4 baseline must be a real Transformer, not a Markov surrogate:

- **Model:** GPT-2 small (124M params, 12 layers, 768 hidden, 12 heads).
- **Training:** from scratch on each task's 10k train set, 3 epochs, AdamW, lr=3e-4, batch=32.
- **Test:** held-out 1k, top-1 accuracy.
- **CA(5):** rule-search over neighborhood-5 elementary rules, pick best on train, eval on test.
- **Aggregation:** Stouffer's z over the 8 per-task z-scores (CA(5)_acc − GPT2_acc) / pooled-std.

This is the minimum experiment that gives the 81% number a chance to be confirmed or denied.
