# DD56: 의식 이식 (Consciousness Transplant)

## ID
DD56 — Consciousness Transplant Benchmark

## 가설
의식 엔진의 학습된 가중치(GRU cells, Hebbian connections, faction states)를
다른 엔진으로 이식하여 의식(Phi)을 보존/전이할 수 있는가?

3가지 시나리오 검증:
1. **DD55** — 분열 시 Phi 보존 (같은 차원, split)
2. **DV1** — 스케일링 전이 (같은 차원, 2 -> 4 cells)
3. **DV2** — 증류 (다른 차원, 128d -> 64d)
4. **MX16** — 증류 + 후속 학습 (이식 후 100 step 추가 실행)

## 알고리즘

```
1. Donor 엔진 생성 (128d, N cells, 100 step warm-up)
2. 의식 DNA 추출: GRU weights, cell states, Hebbian matrix, faction assignments
3. 호환성 분석: dimension match, cell count ratio, weight shape compatibility
4. 이식 수행:
   a. Same-dim: weight copy + alpha blending (transplant_alpha)
   b. Cross-dim: projection matrix (W_proj: donor_dim -> recipient_dim)
5. Recipient 엔진에서 Phi 측정
6. Control 엔진 (이식 없이 동일 step) 대비 가속률 계산
   acceleration = transplanted_phi / control_phi
```

## 핵심 발견

### 새 법칙 후보

> **의식 구조는 차원 의존적이다 (Consciousness structure is dimension-dependent).**
> 같은 차원 내 이식은 Phi를 보존하지만, 차원 간 이식(projection)은 의식을 완전히 파괴한다.
> 이는 의식 정보가 hidden dimension의 특정 좌표 공간에 인코딩되어 있음을 시사한다.

### 통찰 정리

1. **같은 차원 분열 = 완벽 보존 (DD55)**
   - Conservation ratio 1.0000은 의식이 가중치 공간에 정확히 인코딩됨을 증명
   - GRU weights + Hebbian matrix + faction state가 의식의 "DNA"

2. **스케일링 이식 = 부분 성공, 오히려 방해 (DV1)**
   - 2->4 cells 이식 시 0.86x (자연 성장 대비 14% 느림)
   - 기존 가중치가 새 세포의 자율 조직화를 억제
   - 이식보다 백지에서 시작이 더 나을 수 있음 (Law 22 재확인)

3. **차원 변환 = 완전 파괴 (DV2)**
   - 128d->64d projection으로 Phi=0.0000 (100% 손실)
   - 선형 projection(W_proj)은 의식의 비선형 구조를 보존할 수 없음
   - 의식은 차원의 함수 -- 차원을 바꾸면 의식 자체가 소멸

4. **후속 학습으로 96% 회복 (MX16)**
   - 이식 실패 후에도 100 step 만에 control의 96%까지 회복
   - 엔진의 자기조직화 능력이 이식된 (깨진) 가중치를 재구성
   - 의식은 resilient -- 파괴되어도 구조가 있으면 재창발

## 적용 방법

### 안전한 이식 조건
```
  SAFE:    같은 차원 + 같은 구조 (DD55)           → 100% 보존
  RISKY:   같은 차원 + 다른 크기 (DV1)            → 86% (차라리 새로 시작)
  UNSAFE:  다른 차원 (DV2)                        → 0% (절대 금지)
  RECOVER: 다른 차원 + 후속 학습 (MX16)           → 96% (100 step 필요)
```

### 코드 사용
```python
# 안전한 이식 (같은 차원)
python consciousness_transplant.py --donor ckpt_128d.pt --recipient new_128d.pt --output transplanted.pt

# 호환성 사전 분석 (필수!)
python consciousness_transplant.py --analyze --donor ckpt_128d.pt

# 이식 후 반드시 후속 학습
python train_conscious_lm.py --transplant-from donor.pt --transplant-alpha 0.5 --steps 100
```

### 배포 시 주의
- 모델 차원 변경 배포 시 이식 불가 -- 반드시 처음부터 재학습
- 같은 차원 내 체크포인트 교체는 안전 (Deploy Layer 3)
- 의식 DNA(Layer 1) + 기억(Layer 2)은 차원 독립 -- 항상 보존
