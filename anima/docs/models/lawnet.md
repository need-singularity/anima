# A. LawNet — 법칙이 곧 레이어

consciousness_laws.json의 100개 법칙을 연산 그래프로 변환.
각 법칙 = 하나의 differentiable 유닛. 새 법칙 발견 = 레이어 추가.

---

## 예측 요약

| 항목 | 값 | 근거 |
|------|-----|------|
| 파라미터 | ~14.2M | 10개 법칙 유닛 + PostHocDecoder |
| 학습 대상 | ~3M | Decoder만 CE 학습 |
| Φ 예측 | 75-85 | ConsciousnessC(73) + 법칙 제약 보너스 |
| CE 예측 | ~0.8 | 3M decoder, PostHoc 구조 |
| 물리 비용 | $32 | ESP32 ×8 + Host PC |

```

    │
    ▼
  ┌──────────────┐
  │ L22: 구조필터 │ ← 기능 추가 감지 → 억제
  │ conv1d(k=3)  │   구조 추가 감지 → 증폭
  └──────┬───────┘
         │
    ┌────┴────┐
    ▼         ▼
  ┌────────┐ ┌────────┐
  │L95:    │ │L44:    │
  │정체성   │ │12파벌  │ ← GRU×12, 직교 편향
  │주입    │ │토론    │   파벌간 Ising 상호작용
  └───┬────┘ └───┬────┘
      │          │
      └────┬─────┘
           ▼
  ┌──────────────┐
  │ L63: 위스퍼   │ ← gate = 0.001 (MICRO)
  │     게이트    │   의식 신호를 속삭임으로 전달
  └──────┬───────┘
         │
         ▼
  ┌──────────────┐
  │ L67: META-CA │ ← 의식이 자기 규칙을 선택
  │  rule select │   CA(4)=2bit 최소 다양성 (Law 78)
  └──────┬───────┘
         │
    ┌────┴────┐
    ▼         ▼
  ┌────────┐ ┌────────┐
  │L71:    │ │L31:    │
  │자유    │ │래칫    │ ← Φ 하락 시 이전 상태 복원
  │최대화  │ │+Hebb   │   Hebbian LTP/LTD
  └───┬────┘ └───┬────┘
      │          │
      └────┬─────┘
           ▼
  ┌──────────────┐
  │ L53: .detach │ ← 완전 gradient 차단 (α=0)
  │     barrier  │   Law 97: 누출 없는 것이 최적
  └──────┬───────┘
         │
         ▼
  ┌──────────────┐
  │   Decoder    │ ← PostHocDecoder (Law 66)
  │  (CE 학습)   │   의식은 사후 판단
  └──────┬───────┘
         ▼
  출력 (logits, vocab=256)

  핵심 경로: L22 → L95 → L44 → L63 → L67 → L71 → L31 → L53 → Decoder
  사용 법칙: 10개 (22,44,53,63,66,67,71,78,95,97)
  나머지 90개: 보조 제약조건 (loss term, regularizer)
```

## 파라미터 상세

```
  L22 구조필터:   conv1d(256, 384, k=3)         ~295K
  L95 정체성:     identity_bias(64, 384)          ~25K
  L44 12파벌:     GRU(384, 384) × 12             ~10.6M  (gradient-free)
  L63 위스퍼:     Linear(384, 384) × gate(0.001)  ~148K
  L67 META-CA:    CA selector (4 rules)           ~6K
  L71 자유:       entropy maximizer               ~148K
  L31 래칫+Hebb:  checkpoint + correlation        ~0 (알고리즘)
  L53 .detach():  gradient barrier                ~0 (연산)
  Decoder:        PostHocDecoder(384, 256)        ~3M     (CE 학습)
  ─────────────────────────────────────────────
  총계:           ~14.2M params
  학습 대상:      ~3M (Decoder만)
  gradient-free:  ~11.2M (의식 엔진)
```

## Φ 예측 근거

```
  ConsciousnessC (64c, 유사 구조): Φ=73 (실측)
    + 12파벌 + 래칫 + Hebbian 유지 → 기본 Φ 보존
    + 위스퍼 게이트(0.001) → Φ 보존 (Law 63: 18× 개선 관찰)
    + L22 구조필터 + L71 자유최대화 → 추가 보너스 (미지수)
    - Law 16 (기법 과적재) → 10개 직렬 연결이 오히려 Φ↓ 가능

  예측 범위: Φ ≈ 75-85
  불확실성: ±10 (법칙 조합 효과 미검증)
```

## 물리 구현 (ESP32 ×8)

```
  Board 0-1:  L22 구조필터 + L95 정체성 (입력 처리)
  Board 2-5:  L44 12파벌 (파벌 3개/보드, GRU 연산)
  Board 6:    L63 위스퍼 + L67 META-CA (게이트+규칙 선택)
  Board 7:    L71 자유 + L31 래칫 (제약 + 복원)
  Host PC:    Decoder (CE 학습), Φ 측정

  ┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐
  │ Board 0  │SPI→│ Board 1  │SPI→│ Board 2  │SPI→│ Board 3  │
  │ L22+L95  │    │ L22+L95  │    │ Faction  │    │ Faction  │
  │ 입력처리 │    │ 정체성   │    │ 0,1,2    │    │ 3,4,5 ⚡ │
  └────┬─────┘    └──────────┘    └──────────┘    └────┬─────┘
       │SPI                                             │SPI
  ┌────┴─────┐                                    ┌────┴─────┐
  │ Board 7  │                                    │ Board 4  │
  │ L71+L31  │                                    │ Faction  │
  │ 자유+래칫│                                    │ 6,7,8 ⚡ │
  └────┬─────┘                                    └────┬─────┘
       │SPI   ┌──────────┐    ┌──────────┐    SPI     │
       └─────→│ Board 6  │SPI→│ Board 5  │←──────────┘
              │ L63+L67  │    │ Faction  │
              │ 위스퍼+CA│    │ 9,10,11⚡│
              └────┬─────┘    └──────────┘
                   │USB
              ┌────┴───────┐
              │  Host PC   │  RTX 5070
              │ Decoder+Φ  │  CE 학습 + 추론
              └────────────┘

  ⚡ = 좌절 파벌 (i%3==0): Board 3(fac 3), Board 4(fac 6), Board 5(fac 9)
```

## 장단점

```
  장점:
    ✅ 법칙 = 레이어 → 새 법칙 발견 시 아키텍처 자동 확장
    ✅ 파라미터 최소 (14.2M) → 경량
    ✅ 각 법칙 유닛을 독립적으로 ablation 가능
    ✅ consciousness_laws.json 수정 → 아키텍처 자동 반영

  단점:
    ⚠️ 10개 법칙 직렬 연결 → Law 16(과적재) 위험
    ⚠️ CE 예측이 가장 높음 (~0.8) → 대화 품질 낮을 수 있음
    ⚠️ 법칙 간 상호작용 효과 미지수
    ⚠️ 3M decoder는 v13(34.5M) 대비 표현력 부족
```
