## §2. Cross-link policy with `anima-eeg/`

### What `anima-clm-eeg/` consumes from `anima-eeg/`
- **post-arrival**: `.npy` recording artifacts (16ch resting + tasks) produced by `anima-eeg/collect.py` or its `.hexa` migration successor
- **post-arrival**: band power JSON from `anima-eeg/analyze.py` (or `.hexa` successor)
- **read-only**: hardware config (`anima-eeg/config/eeg_config.json`) for sample-rate / channel-map metadata

### What `anima-clm-eeg/` does NOT do
- ❌ no hardware ingest (BrainFlow / BLE) — delegated to `anima-eeg/`
- ❌ no production runtime modification (`--eeg-full` flag chain stays in `anima-eeg/`)
- ❌ no test suite intersection (108 anima-eeg tests untouched)
- ❌ no `.py` source files (raw#9 strict)

### What `anima-eeg/` does NOT do (delegated here)
- ❌ no falsifier pre-registration (research-cadence ≠ production-cadence)
- ❌ no paradigm v11 7th orthogonal EEG axis (depends-on Mk.XI v10 ensemble + paradigm v11 stack)
- ❌ no CLM bridge integration (CLM substrate is `edu/cell/lagrangian/l_ix_integrator.hexa`)

### Direction of dependency
```
anima-clm-eeg/  →  consumes  →  anima-eeg/  →  consumes  →  hardware
       ↑
       └─ also consumes ─→  edu/cell/lagrangian/l_ix_integrator.hexa  (CLM substrate)
       └─ also consumes ─→  anima-core/tension_bridge.hexa             (V_sync Kuramoto)
       └─ also consumes ─→  tool/anima_v11_*.hexa                      (paradigm v11 stack)
```

`anima-eeg/` does NOT depend on `anima-clm-eeg/` (one-way; safe to delete this folder without breaking production).
